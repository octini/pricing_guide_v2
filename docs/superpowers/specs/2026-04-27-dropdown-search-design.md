# Dropdown Search Design

## Summary
Add a case-insensitive, partial-match search input to the three checkbox-based dropdown filter menus (Sourcebook, Item Type, Rarity) on the pricing guide page.

## Design
- Each `.dropdown-content > .checkbox-list` gets a text `<input>` prepended before the checkbox items
- On input, iterate visible `.checkbox-item` siblings and toggle `display: none` based on whether the label text contains the search string (case-insensitive)
- The search is purely a visual filter on the option list — it does not affect filter state, selection, or any other functionality
- All existing behavior (dropdown toggle, checkbox filtering, pagination, tooltips) remains unchanged

## Impact
- Files to modify: `scripts/11_generate_html.py` only (adds search input per dropdown + JS logic)
- Generated output: `index.html` (regenerated by script 11)
- No CSS/JS/files beyond the single HTML generator

## Acceptance Criteria
- Typing in a dropdown search input filters visible options to those containing the text (case-insensitive)
- Clearing the input shows all options again
- Search does not interfere with checkbox selection or other dropdowns
- Styling matches existing dark theme
